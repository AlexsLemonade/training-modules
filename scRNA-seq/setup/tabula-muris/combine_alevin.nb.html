Add sample info to each single cell experiment object. Here we are lso making sure we have a unique cell id that matches the format of TM_droplet_metadata.csv

add_sample_meta <- function(sce, sample_id, meta_table){
  barcodes <- colnames(sce) 
  cell_ids <- paste0(sample_id, "_",  barcodes)
  meta_df <- tibble::tibble(cell = cell_ids, 
                            sample_id = sample_id, 
                            barcode = barcodes) %>%
    dplyr::left_join(meta_table, by = "sample_id")
  colData(sce) <- DataFrame(meta_df, row.names = meta_df$cell)
  return(sce)
}

tm_sces <- purrr::imap(tm_sces, add_sample_meta, tm_metadata)

Filter cells based on QC measures

This may not be ideal, especially for the kidney samples which seem to have high overall mitochondrial percentages, but for now lets set the following cutoffs:

  • Mitochondrial percent < 25%
  • detected genes > 500
filter_sce <- function(sce, detect_threshold, mito_threshold) {
  qc_pass <- ((sce$detected > mito_threshold) 
              & (sce$subsets_mito_percent < mito_threshold))
  sce_filtered <- sce[, qc_pass]
}

tm_filtered <- tm_sces %>%
  purrr::map(filter_sce, detect_threshold = 500, mito_threshold = 25)

Combine SingleCellExperiments

This is a naive combination, simply concatinating the component samples into one SCE class/expression matrix. If we had some batch info, we might want to apply that to do better.

tm_combined <- do.call(cbind, tm_filtered)

Feature (gene) QC

I am doing the Feature QC at this point because it does not use normalized data, and by my understanding of the normalization procedures, removing the low-expressed genes should not affect normalization.

tm_combined <- addPerFeatureQC(tm_combined)

Feature filtering criteria: - Genes found in at least 10 samples - mean count of at least 0.5

detected <- rowData(tm_combined)$detected > 10
expressed <- rowData(tm_combined)$mean > 0.5

# filter the genes (rows) this time
tm_combined <- tm_combined[detected & expressed, ]

Normalization

We are normalizing with scran::computeSumFactors() which implements the strategy of Lun et al. (2016).

# Cluster similar cells
qclust <- scran::quickCluster(tm_combined)

# Step 2) Compute sum factors for each cell cluster grouping.  
tm_combined <- scran::computeSumFactors(tm_combined, clusters = qclust)

# Step 3) Normalize using these pooled sum factors and log transform. 
tm_combined <- scater::logNormCounts(tm_combined)

Variance filtering

Since we are using this as an example data set, we are going to shrink it down past what we might normally do. We will select the 1000 genes with the highest biologial variance across samples, after modeling the technical variance using scran::modelGeneVar()

gene_var <- scran::modelGeneVar(tm_combined)
highvar_genes <- scran::getTopHVGs(gene_var, n = 1000)

# filter
tm_combined_filt <- tm_combined[highvar_genes,]

Write out the SCE object as an RDS for later use.

readr::write_rds(tm_combined_filt, outfile)
